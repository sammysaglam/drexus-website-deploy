Not all user flows deserve the same latency budget. Our analysis of 500+ production systems reveals
the P95 latency targets that balance user experience with engineering reality. These benchmarks
represent what's actually achievable, not theoretical ideals.

<Callout type="info" title="Why P95 Matters">
  P95 latency (95th percentile) captures the experience of your unluckiest regular users. It's the
  metric that correlates most strongly with user satisfaction and business outcomes.
</Callout>

## Executive Summary: The 3-3-5 Rule

For most applications, remember 3-3-5:

- **3 seconds** for authentication flows
- **3 seconds** for search operations
- **5 seconds** for checkout completion

These P95 targets achieve the optimal balance between user satisfaction (&gt;4.2/5 rating) and
implementation cost.

| Username/Password     | 450ms   | 1,800ms | 3,200ms | &lt;1,000ms  | Includes bcrypt/argon2 |

| Magic Link            | 350ms   | 1,200ms | 2,800ms | &lt;800ms    | Just email send        |
| Biometric             | 150ms   | 600ms   | 1,400ms | &lt;400ms    | Device-side only       |
| 2FA Verification      | 200ms   | 800ms   | 1,900ms | &lt;500ms    | TOTP calculation       |

### Budget Breakdown: Password Authentication

Here's how the 1,800ms P95 budget typically allocates:

```
Total Budget: 1,800ms
├── Network RTT (2x): 100ms (6%)
├── TLS Handshake: 150ms (8%)
├── Database Query: 200ms (11%)
├── Password Hashing: 250ms (14%)
├── Session Creation: 150ms (8%)
├── JWT Generation: 100ms (6%)
├── Response Serialization: 50ms (3%)
├── Frontend Processing: 300ms (17%)
└── Buffer/Variance: 500ms (27%)
```

<Aside>
  **Pro tip**: If your auth flow exceeds 3s P95, users perceive your entire app as slow. This "halo
  effect" impacts satisfaction scores by up to 35%.
</Aside>

### Setting Your Budgets

Don't copy benchmarks blindly. Adjust based on:

1. **User Expectations**: B2B tolerates higher latency than B2C
2. **Network Reality**: International users need larger budgets
3. **Business Model**: Subscription services can be slower than e-commerce
4. **Competition**: Match or beat your direct competitors

### Progressive Budget Improvement

Start where you are, improve incrementally:

<Step number={1} title="Baseline Current P95">
  Measure your actual P95 for each flow. No judgment, just data. This is your starting point.
</Step>

<Step number={2} title="Set 3-Month Target">
  Aim for 20% improvement. If you're at 6s, target 4.8s. Achievable goals drive action.
</Step>

<Step number={3} title="Identify Biggest Wins">
  Use waterfall analysis. Fix the largest contributors first. Usually 2-3 changes drive 80%
  improvement.
</Step>

<Step number={4} title="Monitor and Iterate">
  Track weekly. Celebrate improvements. Set new targets quarterly. Continuous improvement compounds.
</Step>
